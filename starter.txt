==================================================

Build a practical internal dashboard so the client can control the site without editing code.

The dashboard should allow management of:

Editable content across all pages:
- headings
- subheadings
- paragraphs
- CTA labels
- CTA links
- section visibility (optional if simple enough)
- contact details
- about us content
- homepage content blocks
- catalog intro content
- product detail supporting content

### B. Products
A proper product management module:
- add product
- edit product
- delete product
- publish/unpublish product
- slug
- product title
- short description
- long description
- specifications
- features
- applications
- category
- sort order
- featured image
- gallery images
- SEO title
- SEO description
- SEO keywords or equivalent metadata
- OG image if feasible
- status

### C. Media Library
A backend media manager with:
- upload media
- image listing/gallery
- file preview where possible
- file metadata
- image reuse across sections
- delete media
- select existing media for pages/products
- safe upload validation

### D. SEO Management
Need a centralized way to manage:
- per-page meta title
- per-page meta description
- canonical URL if needed
- social sharing image
- robots meta if needed
- schema / JSON-LD fields if feasible
- global SEO defaults
- header scripts
- footer scripts

### E. Header/Footer Script Injection
Need editable fields in dashboard for:
- custom HTML in head
- custom scripts in head
- footer scripts
- analytics scripts
- verification tags
- other header/footer code insertions

Sanitize carefully but still allow intended admin control.

### F. Theme / Site Settings
Need a settings area for:
- site title
- logo
- favicon
- contact info
- address
- social links
- primary theme color(s)
- secondary theme color(s)
- CTA button text defaults if needed
- global branding assets

Important:
- Theme settings should update colors/branding without changing layout structure.
- The layout/design must remain locked.
- Only theme variables and editable assets/content should be changeable.

### G. User Management
Need basic user management with 2 roles for now:
1. Admin
2. Content Editor (non-admin)

Permissions:
- Admin:
  - full access
  - manage users
  - add/remove users
  - manage settings
  - manage SEO
  - manage scripts
  - manage all content and products

- Content Editor:
  - edit content
  - edit products
  - upload/select media
  - update text/images/content
  - cannot manage users
  - cannot delete or create admin users
  - should not have access to sensitive system settings unless explicitly safe

Keep RBAC simple and clear.

PHASE 6 — TECHNICAL EXPECTATIONS
==================================================

Please use a clean PHP architecture that is practical for shared hosting.

Recommendations:
- PHP 8+
- MySQL / MariaDB
- Apache compatible
- `.env` driven configuration
- PDO for database access
- secure authentication/session handling
- CSRF protection for admin forms
- password hashing using PHP built-ins
- validation and sanitization throughout
- flash messages for dashboard actions
- file upload restrictions and validation
- prepared statements everywhere
